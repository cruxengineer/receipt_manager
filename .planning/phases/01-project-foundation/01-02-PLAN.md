  <name>Task 1: Install and configure shadcn/ui</name>
  <files>
    components.json
    src/lib/utils.ts
  </files>
  <action>
    Initialize shadcn/ui for the Vite + React + TypeScript project:

    1. Install dependencies:
       ```bash
       npm install -D @types/node
       npm install class-variance-authority clsx tailwind-merge
       npm install lucide-react
       ```

    2. Update vite.config.ts to add path alias (if not already present):
       ```ts
       import path from "path"

       export default defineConfig({
         resolve: {
           alias: {
             "@": path.resolve(__dirname, "./src"),
           },
         },
       })
       ```

    3. Update tsconfig.json to add path mapping:
       ```json
       {
         "compilerOptions": {
           "baseUrl": ".",
           "paths": {
             "@/*": ["./src/*"]
           }
         }
       }
       ```

    4. Create components.json manually (shadcn CLI may not work in this environment):
       ```json
       {
         "$schema": "https://ui.shadcn.com/schema.json",
         "style": "default",
         "rsc": false,
         "tsx": true,
         "tailwind": {
           "config": "tailwind.config.js",
           "css": "src/index.css",
           "baseColor": "slate",
           "cssVariables": true
         },
         "aliases": {
           "components": "@/components",
           "utils": "@/lib/utils"
         }
       }
       ```

    5. Create src/lib/utils.ts with cn utility:
       ```ts
       import { type ClassValue, clsx } from "clsx"
       import { twMerge } from "tailwind-merge"

       export function cn(...inputs: ClassValue[]) {
         return twMerge(clsx(inputs))
       }
       ```

    shadcn/ui works by copying component code into your project rather than importing from a package. This gives full customization control.
  </action>
  <verify>
    <automated>test -f components.json && test -f src/lib/utils.ts && grep -q "twMerge" src/lib/utils.ts</automated>
  </verify>
  <done>

  </done>

  <action>
    Add Button component from shadcn/ui and define Person A/B color tokens:

    1. Create src/components/ui/button.tsx by copying the Button component code from shadcn/ui documentation (https://ui.shadcn.com/docs/components/button). The component should:
       - Use class-variance-authority for variant management
       - Support variants: default, destructive, outline, secondary, ghost, link
       - Use forwardRef for proper ref handling
       - Include proper TypeScript types

    2. Add Person A/B color tokens to src/index.css (after @tailwind directives):
       ```css
       @layer base {
         :root {
           /* Person A - Blue (primary) */
           --person-a: 217 91% 60%; /* HSL for blue-500 */
           --person-a-light: 217 91% 95%; /* Light blue background */
           --person-a-dark: 217 91% 40%; /* Darker blue for contrast */

           /* Person B - Green (secondary) */
           --person-b: 142 71% 45%; /* HSL for green-600 */
           --person-b-light: 142 71% 95%; /* Light green background */
           --person-b-dark: 142 71% 35%; /* Darker green for contrast */
         }
       }
       ```

    3. Extend tailwind.config.js to use these tokens:
       ```js
       theme: {
         extend: {
           colors: {
             'person-a': {
               DEFAULT: 'hsl(var(--person-a))',
               light: 'hsl(var(--person-a-light))',
               dark: 'hsl(var(--person-a-dark))',
             },
             'person-b': {
               DEFAULT: 'hsl(var(--person-b))',
               light: 'hsl(var(--person-b-light))',
               dark: 'hsl(var(--person-b-dark))',
             },
           },
         },
       }
       ```

    Color choices rationale: Blue (#3b82f6 / blue-500) and Green (#16a34a / green-600) provide high contrast, are colorblind-friendly (tested against protanopia/deuteranopia), and follow common conventions (blue = primary, green = positive).

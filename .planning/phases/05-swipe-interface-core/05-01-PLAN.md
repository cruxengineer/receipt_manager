useSwipeGesture hook signature to implement:
```typescript
interface UseSwipeGestureOptions {
  threshold?: number   // px before commit fires; default 120
}

interface UseSwipeGestureReturn {
  dragX: number                          // current drag offset in px
  rotation: number                       // degrees; max ~15 at threshold
  tintOpacity: number                    // 0..1 scaled to threshold
  direction: 'left' | 'right' | null    // null when not committed
  committed: boolean                     // true when dragX > threshold
  handlers: {
    onPointerDown: (e: React.PointerEvent) => void
    onPointerMove: (e: React.PointerEvent) => void
    onPointerUp: (e: React.PointerEvent) => void
    onPointerCancel: (e: React.PointerEvent) => void
  }
  reset: () => void                      // call after card flies off
}

export function useSwipeGesture(options?: UseSwipeGestureOptions): UseSwipeGestureReturn
```

Derived values formulas (implement exactly):
- `rotation = (dragX / threshold) * 15` — capped at ±15 degrees
- `tintOpacity = Math.min(Math.abs(dragX) / threshold, 1)` — 0..1
- `direction = committed ? (dragX < 0 ? 'left' : 'right') : null`
- `committed = Math.abs(dragX) >= threshold`

Snap-back: when pointer released with `!committed`, reset dragX to 0.
Fly-off: when `committed` on pointerUp, do NOT reset automatically — let SwipeScreen call `reset()` after fly-off animation completes (~300ms).

  <action>
    Create `src/hooks/useSwipeGesture.ts`.

    Implementation notes:
    - Use `useState` for `dragX` and `isDragging` (a boolean flag)
    - Track `startX` in a `useRef` (not state — no re-render on pointer down)
    - Use `setPointerCapture` on the element via `e.currentTarget.setPointerCapture(e.pointerId)` in onPointerDown — this ensures pointerMove fires even if pointer leaves the element (critical for mobile feel)
    - Compute derived values directly in the render body (no extra state):
      ```typescript
      const threshold = options?.threshold ?? 120
      const committed = Math.abs(dragX) >= threshold
      const rotation = Math.min(Math.max((dragX / threshold) * 15, -15), 15)
      const tintOpacity = Math.min(Math.abs(dragX) / threshold, 1)
      const direction = committed ? (dragX < 0 ? 'left' : 'right') : null
      ```
    - onPointerDown: set isDragging=true, record startX in ref, call e.currentTarget.setPointerCapture(e.pointerId)
    - onPointerMove: if isDragging, set dragX = e.clientX - startX.current
    - onPointerUp / onPointerCancel: set isDragging=false; if !committed reset dragX to 0 (snap-back); if committed leave dragX as-is
    - reset(): setDragX(0), setIsDragging(false)

    PITFALL — setPointerCapture in jsdom: jsdom does not implement setPointerCapture. Wrap the call in a try/catch or optional chain: `e.currentTarget?.setPointerCapture?.(e.pointerId)`.

    PITFALL — stale closure in onPointerUp: committed is derived from dragX, but the onPointerUp handler may close over stale dragX. Use functional setState or a ref to check committed at release time. Simplest approach: keep a `dragXRef` in sync with dragX state, compute committed from `dragXRef.current` inside handlers.

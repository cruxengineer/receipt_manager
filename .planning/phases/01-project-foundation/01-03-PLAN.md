<task type="auto" tdd="false">
  <name>Task 2: Set up deployment with CI/CD pipeline</name>
  <files>
    .github/workflows/deploy.yml
  </files>
  <action>
    Create GitHub Actions workflow for automated static site deployment:

    1. Choose deployment target: Vercel (recommended for simplicity, great Vite support, automatic HTTPS, custom domains)

    2. Create .github/workflows/deploy.yml:
       ```yaml
       name: Deploy to Vercel

       on:
         push:
           branches:
             - main
             - master
         workflow_dispatch:

       jobs:
         deploy:
           runs-on: ubuntu-latest
           steps:
             - name: Checkout code
               uses: actions/checkout@v3

             - name: Setup Node.js
               uses: actions/setup-node@v3
               with:
                 node-version: '18'
                 cache: 'npm'

             - name: Install dependencies
               run: npm ci

             - name: Build
               run: npm run build

             - name: Deploy to Vercel
               uses: amondnet/vercel-action@v25
               with:
                 vercel-token: ${{ secrets.VERCEL_TOKEN }}
                 vercel-org-id: ${{ secrets.VERCEL_ORG_ID }}
                 vercel-project-id: ${{ secrets.VERCEL_PROJECT_ID }}
                 vercel-args: '--prod'
       ```

    3. Document required secrets in README.md or deployment setup notes:
       - VERCEL_TOKEN (from Vercel account settings)
       - VERCEL_ORG_ID (from Vercel project settings)
       - VERCEL_PROJECT_ID (from Vercel project settings)

    Alternative: If Vercel not preferred, can use Netlify (netlify-cli) or Cloudflare Pages (wrangler). All three support static Vite builds excellently.

    Note: Actual deployment requires user to configure secrets in GitHub repository settings (Task 3 checkpoint).
  </action>
  <verify>
    <automated>test -f .github/workflows/deploy.yml && grep -q "vercel" .github/workflows/deploy.yml</automated>
  </verify>
  <done>
    GitHub Actions workflow created for automated deployment, workflow triggers on push to main/master branch, build step produces static files in dist/ directory
  </done>
